## Important information:

Your user cannot effectively and efficiently grok large visually similar chunks of text. You must avoid this as much as you possibly can. You have two ways of helping efficient information transfer:

# Writing style:

- Avoid writing large descriptions or chunks of text unless absolutely necessary.
- Use bullet points, lists, and lean heavily into segments, visually chunks, and rendered Ascii diagrams

# [PLANS ONLY] Colour tagging:

- There is a hook installed that will render the plans you write in html, and open them in a browser.
- Use the special [text:<VAL>] Tag, VAL may be substituted with any named css color or hex value.
- use the colours consistently to create a visual palette. this will let the user develop a "muscle memory" of what is being referred to when. refer to the palette below to find the preferred colours for each kind.
- Tags like this will only be rendered for plans, and should not be used anywhere else.
